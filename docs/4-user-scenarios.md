## 1. 개요

### 1.1 문서 목적

이 문서는 cjy-todoList 애플리케이션의 실제 사용 시나리오를 구체적으로 기술합니다. 각 페르소나가 실제로 시스템을 어떻게 사용하는지 단계별로 설명하여, 개발팀이 사용자 중심의 기능을 구현할 수 있도록 돕습니다.

### 1.2 페르소나 요약

#### 페르소나 1: 김철수 (직장인)
- **연령/직업**: 30대 / 회사원 (마케팅팀)
- **기술 숙련도**: 중급 (일반적인 웹 애플리케이션 사용 가능)
- **주요 니즈**: 업무와 개인 일정을 한곳에서 간단하게 관리
- **사용 환경**: 주로 데스크톱, 가끔 모바일

#### 페르소나 2: 김민정 (대학생)
- **연령/직업**: 20대 / 대학생 (컴퓨터공학과 3학년)
- **기술 숙련도**: 고급 (다양한 디지털 도구 사용 경험)
- **주요 니즈**: 과제, 시험, 팀 프로젝트 등 학업 일정 관리
- **사용 환경**: 데스크톱과 모바일 병행

## 2. 주요 사용자 시나리오

### 시나리오 1: 첫 사용자의 회원가입 및 첫 할 일 등록

**페르소나**: 김철수 (직장인)

**상황**:
김철수는 여러 할 일 관리 앱을 사용해봤지만, 너무 복잡하거나 기능이 많아 사용하기 번거로웠습니다. 동료의 추천으로 cjy-todoList를 알게 되었고, 간단하고 가볍다는 설명에 시도해보기로 결정했습니다.

**목표**:
- 계정을 생성하고 로그인하기
- 첫 번째 할 일 등록하기
- 기본 기능 파악하기

**시나리오 단계**:

#### 1단계: 웹사이트 접속 및 회원가입

1. 김철수는 브라우저에서 cjy-todoList 웹사이트에 접속합니다.
2. 메인 페이지에서 "회원가입" 버튼을 클릭합니다.
3. 회원가입 폼이 표시됩니다:
   - 사용자명: `chulsoo_kim`
   - 이메일: `chulsoo.kim@company.com`
   - 비밀번호: `SecurePass123!`
   - 비밀번호 확인: `SecurePass123!`
4. "회원가입" 버튼을 클릭합니다.
5. **시스템**: 입력값을 검증하고 계정을 생성합니다.
6. **시스템**: "회원가입이 완료되었습니다. 로그인해주세요." 메시지를 표시합니다.
7. 자동으로 로그인 페이지로 이동합니다.

**예상 결과**:
- ✅ 계정이 성공적으로 생성됨
- ✅ 로그인 페이지로 리다이렉트됨
- ✅ 환영 메시지 표시

#### 2단계: 로그인

1. 로그인 폼에 정보를 입력합니다:
   - 사용자명/이메일: `chulsoo_kim`
   - 비밀번호: `SecurePass123!`
2. "로그인" 버튼을 클릭합니다.
3. **시스템**: 인증 정보를 확인하고 JWT 토큰을 발급합니다.
4. **시스템**: 메인 대시보드로 리다이렉트합니다.

**예상 결과**:
- ✅ 로그인 성공
- ✅ Access Token과 Refresh Token 저장됨
- ✅ 메인 대시보드 표시 (빈 할 일 목록)

#### 3단계: 첫 번째 할 일 추가

1. 메인 대시보드에서 "할 일 추가" 버튼을 클릭합니다.
2. 할 일 추가 폼이 표시됩니다.
3. 다음 정보를 입력합니다:
   - 제목: `분기 보고서 작성`
   - 설명: `2025년 1분기 마케팅 성과 보고서 작성 및 발표 자료 준비`
   - 시작일: `2025-11-25`
   - 마감일: `2025-11-30`
4. "저장" 버튼을 클릭합니다.
5. **시스템**: 할 일을 데이터베이스에 저장합니다.
6. **시스템**: 할 일 목록으로 리다이렉트하고 방금 추가한 항목을 표시합니다.

**예상 결과**:
- ✅ 할 일이 성공적으로 생성됨
- ✅ 목록에 새 할 일이 표시됨
- ✅ "할 일이 추가되었습니다" 확인 메시지

#### 4단계: 더 많은 할 일 추가

김철수는 계속해서 다음 할 일들을 추가합니다:

**할 일 2**:
- 제목: `팀 미팅 준비`
- 설명: `주간 팀 미팅 안건 정리`
- 마감일: `2025-11-26`

**할 일 3**:
- 제목: `고객사 이메일 답변`
- 설명: `A사 문의사항 답변 작성`
- 마감일: `2025-11-25`

**예상 결과**:
- ✅ 3개의 할 일이 목록에 표시됨
- ✅ 마감일순으로 정렬되어 있음

### 시나리오 2: 일상적인 할 일 관리

**페르소나**: 김철수 (직장인)

**상황**:
김철수는 이제 매일 아침 출근 후 cjy-todoList를 열어 오늘의 할 일을 확인하는 것이 습관이 되었습니다. 오늘은 월요일 아침이고, 한 주를 시작하며 할 일을 정리하려고 합니다.

**목표**:
- 오늘의 할 일 확인하기
- 완료한 작업 체크하기
- 새로운 할 일 추가하기
- 일정 조정하기

**시나리오 단계**:

#### 1단계: 로그인 및 오늘의 할 일 확인

1. 김철수는 웹사이트에 접속하고 자동 로그인됩니다 (토큰이 유효함).
2. 메인 대시보드에서 오늘의 할 일 목록을 확인합니다.
3. **시스템**: 마감일이 오늘인 할 일 3개를 하이라이트하여 표시합니다:
   - ⚠️ `고객사 이메일 답변` (오늘 마감)
   - `팀 미팅 준비` (내일 마감)
   - `분기 보고서 작성` (5일 후 마감)

**예상 결과**:
- ✅ 오늘 마감인 할 일이 시각적으로 강조됨
- ✅ 마감일이 임박한 순서로 정렬됨

#### 2단계: 할 일 완료 처리

1. 김철수는 아침 일찍 `고객사 이메일 답변`을 완료했습니다.
2. 해당 할 일 옆의 체크박스를 클릭합니다.
3. **시스템**: 할 일의 `isCompleted` 상태를 `true`로 업데이트합니다.
4. **시스템**: 할 일에 취소선이 그어지고 색상이 회색으로 변경됩니다.

**예상 결과**:
- ✅ 할 일이 완료 상태로 표시됨
- ✅ 시각적으로 완료되었음을 명확히 알 수 있음

#### 3단계: 긴급 할 일 추가

1. 팀장으로부터 긴급 요청이 들어왔습니다.
2. "할 일 추가" 버튼을 클릭합니다.
3. 다음 정보를 입력합니다:
   - 제목: `긴급: 클라이언트 프레젠테이션 수정`
   - 설명: `B사 요청사항 반영하여 PT 자료 수정`
   - 마감일: `2025-11-25` (오늘)
4. "저장" 버튼을 클릭합니다.
5. **시스템**: 새 할 일을 추가하고 마감일이 오늘이므로 상단에 표시합니다.

**예상 결과**:
- ✅ 긴급 할 일이 목록 최상단에 표시됨
- ✅ 오늘 마감 표시가 강조됨

#### 4단계: 할 일 수정 (일정 연기)

1. 김철수는 긴급 업무 때문에 `분기 보고서 작성`을 조금 미뤄야 한다고 판단했습니다.
2. 해당 할 일을 클릭하여 상세 페이지로 이동합니다.
3. "수정" 버튼을 클릭합니다.
4. 마감일을 `2025-11-30`에서 `2025-12-02`로 변경합니다.
5. "저장" 버튼을 클릭합니다.
6. **시스템**: 할 일 정보를 업데이트합니다.
7. **시스템**: 목록으로 돌아가고 업데이트된 마감일이 반영됩니다.

**예상 결과**:
- ✅ 마감일이 성공적으로 변경됨
- ✅ 목록에서 정렬 순서가 변경됨

### 시나리오 3: 캘린더 뷰를 활용한 월간 일정 관리

**페르소나**: 김민정 (대학생)

**상황**:
김민정은 이번 주가 중간고사 기간이고, 여러 과목의 시험과 과제 마감이 겹쳐있습니다. 한눈에 일정을 파악하기 위해 캘린더 뷰를 사용하려고 합니다.

**목표**:
- 캘린더에서 이번 달 전체 일정 확인하기
- 시험 및 과제 일정 추가하기
- 국경일 확인하여 계획 조정하기

**시나리오 단계**:

#### 1단계: 캘린더 뷰 접근

1. 김민정은 로그인 후 상단 네비게이션에서 "캘린더" 메뉴를 클릭합니다.
2. **시스템**: 현재 월(11월)의 캘린더를 표시합니다.
3. 캘린더에는 다음 정보가 표시됩니다:
   - 각 날짜
   - 해당 날짜의 할 일 개수 (점으로 표시)
   - 국경일 (빨간색으로 표시)

**예상 결과**:
- ✅ 월간 캘린더가 표시됨
- ✅ 할 일이 있는 날짜에 시각적 표시가 있음
- ✅ 국경일이 구분되어 표시됨

#### 2단계: 시험 일정 등록

김민정은 이번 달 시험 일정을 한꺼번에 등록합니다.

**시험 1**:
1. 11월 27일 칸을 클릭합니다.
2. "할 일 추가" 팝업이 나타납니다.
3. 정보를 입력합니다:
   - 제목: `중간고사 - 자료구조`
   - 설명: `Chapter 1-5 범위, 트리와 그래프 중심`
   - 마감일: `2025-11-27`
4. "저장" 버튼을 클릭합니다.

같은 방식으로 다음 시험들을 추가합니다:
- 11/28: `중간고사 - 운영체제`
- 11/29: `중간고사 - 데이터베이스`
- 12/01: `중간고사 - 네트워크`

**예상 결과**:
- ✅ 각 날짜에 시험 일정이 표시됨
- ✅ 캘린더에서 시험이 몰려있는 기간을 한눈에 파악 가능

#### 3단계: 과제 마감일 등록

1. 11월 26일을 클릭합니다.
2. 할 일을 추가합니다:
   - 제목: `알고리즘 과제 제출`
   - 설명: `동적 프로그래밍 문제 5개 풀이`
   - 시작일: `2025-11-20`
   - 마감일: `2025-11-26`

**예상 결과**:
- ✅ 과제가 시작일부터 마감일까지 기간으로 표시됨
- ✅ 시험 전에 과제를 마쳐야 함을 시각적으로 파악

#### 4단계: 국경일 고려한 계획 수립

1. 김민정은 캘린더에서 12월 25일이 크리스마스로 표시된 것을 확인합니다.
2. 해당 기간에 팀 프로젝트가 예정되어 있었으나, 휴일을 고려하여 일정을 조정하기로 결정합니다.
3. 12월 25일에 예정된 `팀 프로젝트 회의` 할 일을 클릭합니다.
4. 마감일을 12월 23일로 변경합니다.

**예상 결과**:
- ✅ 국경일 정보가 일정 계획에 도움이 됨
- ✅ 휴일을 고려한 현실적인 계획 수립

#### 5단계: 특정 날짜 상세 보기

1. 김민정은 11월 27일을 클릭하여 해당 날짜의 모든 할 일을 확인합니다.
2. **시스템**: 11월 27일의 할 일 목록을 팝업 또는 사이드바로 표시합니다:
   - `중간고사 - 자료구조`
   - `스터디 모임` (이전에 추가한 일정)
3. 김민정은 시험이 있는 날 스터디 모임이 예정되어 있음을 확인하고 일정을 조정합니다.

**예상 결과**:
- ✅ 특정 날짜의 모든 일정을 한눈에 확인
- ✅ 일정 충돌 파악 및 조정 가능

### 시나리오 4: 실수로 삭제한 할 일 복구

**페르소나**: 김철수 (직장인)

**상황**:
김철수는 완료된 할 일들을 정리하다가 실수로 아직 완료되지 않은 중요한 할 일을 삭제했습니다. 다행히 휴지통 기능이 있다는 것을 기억하고 복구하려고 합니다.

**목표**:
- 삭제된 할 일 찾기
- 할 일 복구하기

**시나리오 단계**:

#### 1단계: 할 일 실수로 삭제

1. 김철수는 할 일 목록에서 `분기 보고서 작성` 항목 옆의 삭제 버튼을 클릭합니다.
2. **시스템**: "이 할 일을 삭제하시겠습니까? (휴지통으로 이동됩니다)" 확인 대화상자를 표시합니다.
3. 김철수는 잘못 클릭한 것을 인지하지 못하고 "확인" 버튼을 클릭합니다.
4. **시스템**: 할 일의 `isDeleted`를 `true`로 설정하고 `deletedAt`에 현재 시간을 기록합니다.
5. **시스템**: 할 일이 목록에서 사라집니다.
6. 김철수는 중요한 할 일이 사라진 것을 깨닫고 당황합니다.

**예상 결과**:
- ✅ 할 일이 목록에서 제거됨
- ✅ 데이터는 데이터베이스에 보존됨 (소프트 삭제)

#### 2단계: 휴지통 접근

1. 김철수는 상단 네비게이션에서 "휴지통" 메뉴를 클릭합니다.
2. **시스템**: 삭제된 모든 할 일 목록을 표시합니다.
3. 목록에는 다음 정보가 표시됩니다:
   - 할 일 제목
   - 원래 마감일
   - 삭제된 날짜
   - 복구 버튼
   - 영구 삭제 버튼

**예상 결과**:
- ✅ 휴지통 페이지가 열림
- ✅ 삭제된 할 일 목록이 표시됨

#### 3단계: 할 일 복구

1. 김철수는 휴지통에서 `분기 보고서 작성` 항목을 찾습니다.
2. 해당 항목의 "복구" 버튼을 클릭합니다.
3. **시스템**: "이 할 일을 복구하시겠습니까?" 확인 대화상자를 표시합니다.
4. "확인" 버튼을 클릭합니다.
5. **시스템**:

4. **시스템**: "만료일은 시작일 이전일 수 없습니다." 오류 메시지를 표시합니다.

**예상 결과**:
- ✅ 비즈니스 규칙(C-12) 적용
- ✅ 논리적 오류 방지
- ❌ 할 일 생성 실패

---

### 예외 시나리오 5: 세션 만료

**상황**: 김철수가 오랫동안 페이지를 열어두었다가 작업하려고 하니 세션이 만료되었습니다.

**단계**:
1. 김철수는 2시간 동안 다른 일을 하다가 돌아왔습니다.
2. 할 일을 완료 처리하려고 체크박스를 클릭합니다.
3. **시스템**: API 요청을 보냅니다.
4. **시스템**: Access Token이 만료된 것을 확인합니다.
5. **시스템**: Refresh Token으로 새 Access Token을 발급받으려고 시도합니다.
6. **두 가지 경우**:

   **경우 A - Refresh Token 유효**:
   - 새 Access Token을 발급받습니다.
   - 원래 요청을 재시도합니다.
   - 할 일이 완료 처리됩니다.
   - 사용자는 중단 없이 작업을 계속합니다.

   **경우 B - Refresh Token도 만료**:
   - "세션이 만료되었습니다. 다시 로그인해주세요." 메시지를 표시합니다.
   - 로그인 페이지로 리다이렉트합니다.
   - 로그인 후 원래 페이지로 돌아갑니다.

**예상 결과**:
- ✅ 자동 토큰 갱신 (경우 A)
- ✅ 명확한 세션 만료 안내 (경우 B)

---

### 예외 시나리오 6: 완료된 할 일 수정 시도

**상황**: 김철수가 이미 완료된 할 일을 수정하려고 시도합니다.

**단계**:
1. 김철수는 완료된 `고객사 이메일 답변` 할 일을 클릭합니다.
2. "수정" 버튼을 클릭합니다.
3. **시스템**: "완료된 할 일은 수정할 수 없습니다." 오류 메시지를 표시합니다.
4. **시스템**: "수정하려면 먼저 완료 상태를 해제하세요." 안내 메시지를 추가로 표시합니다.

**예상 결과**:
- ✅ 비즈니스 규칙(C-15) 적용
- ✅ 해결 방법 안내
- ❌ 수정 불가

**해결 방법**:
1. 김철수는 완료 체크박스를 클릭하여 완료 상태를 해제합니다.
2. 이제 "수정" 버튼이 활성화됩니다.
3. 할 일을 수정할 수 있습니다.

---

### 예외 시나리오 7: 네트워크 오류

**상황**: 김민정이 할 일을 추가하는 중에 인터넷 연결이 끊겼습니다.

**단계**:
1. 김민정은 할 일 추가 폼에 정보를 입력합니다.
2. "저장" 버튼을 클릭합니다.
3. **시스템**: API 요청을 보냅니다.
4. **시스템**: 네트워크 오류를 감지합니다.
5. **시스템**: "네트워크 오류가 발생했습니다. 인터넷 연결을 확인해주세요." 오류 메시지를 표시합니다.
6. 입력한 데이터는 폼에 유지됩니다.

**예상 결과**:
- ✅ 명확한 오류 원인 안내
- ✅ 입력 데이터 보존
- ❌ 할 일 생성 실패

**해결 방법**:
1. 김민정은 인터넷 연결을 복구합니다.
2. "저장" 버튼을 다시 클릭합니다.
3. 할 일이 성공적으로 저장됩니다.

## 5. 성공 기준

각 시나리오의 성공 여부는 다음 기준으로 평가됩니다:

### 5.1 효율성
- ⏱️ **목표 달성 시간**: 주요 작업을 3클릭 이내에 완료
- 🚀 **응답 시간**: 모든 페이지가 2초 이내에 로드
- 💨 **할 일 추가 시간**: 30초 이내에 새 할 일 추가 가능

### 5.2 정확성
- ✅ **데이터 무결성**: 100% 정확한 데이터 저장 및 조회
- 🔒 **보안**: 권한 없는 접근 0건
- 🛡️ **오류 방지**: 입력 유효성 검사로 오류 사전 방지

### 5.3 만족도
- 😊 **사용 편의성**: 도움말 없이 주요 기능 사용 가능
- 🎯 **목표 달성**: 사용자가 원하는 작업 100% 완료
- 💡 **직관성**: 3번 이내 시도로 원하는 기능 찾기

### 5.4 안정성
- 🔄 **복구 가능성**: 실수로 삭제한 데이터 100% 복구
- 📱 **접근성**: 데스크톱과 모바일 모두 지원
- ⚡ **가용성**: 99.5% 이상 시스템 가동

- **세션 관리**: 적절한 토큰 만료 시간 설정
- **입력 검증**: XSS, SQL Injection 방어
