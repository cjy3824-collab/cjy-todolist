## 1. 제품 개요

### 1.1 제품 소개

**cjy-todoList**는 사용자 인증 기반의 개인 할 일 관리 웹 애플리케이션입니다. 사용자는 자신의 할 일을 체계적으로 생성, 관리하고, 캘린더 뷰를 통해 시각화할 수 있으며, 국경일 정보와 통합하여 일정을 한눈에 파악할 수 있습니다.

### 1.2 제품 비전

사용자들이 일상의 할 일을 효율적으로 관리하고 생산성을 높일 수 있도록 돕는 것을 목표로 합니다. 간단하고 직관적이며 편리한 가벼운 애플리케이션을 지향합니다.

### 1.3 핵심 가치 제안 (Value Proposition)

- **간단함**: 복잡하지 않은 직관적인 인터페이스
- **편리함**: 빠른 할 일 추가 및 관리
- **안전함**: 실수로 삭제한 할 일 복구 기능
- **시각화**: 캘린더를 통한 일정 한눈에 보기
- **통합**: 국경일과 개인 일정을 함께 관리

### 1.4 비즈니스 목표

- **주요 목표**: 사용자 기반 확보
- **부가 목표**: 지속적인 사용자 참여 유도

## 2. 타겟 사용자

### 2.1 주요 타겟 사용자

- **개인 사용자**: 일상 생활의 할 일을 관리하고자 하는 일반 사용자
- **직장인**: 업무와 개인 일정을 함께 관리하고자 하는 사용자
- **학생**: 학업 일정, 과제, 시험 등을 관리하고자 하는 사용자

### 2.2 사용자 페르소나

#### 페르소나 1: 김철수 (직장인)

- **나이/직업**: 30대 / 직장인
- **목표**: 업무와 개인 일정을 하나의 플랫폼에서 간단하게 관리
- **페인 포인트**:
  - 복잡한 일정 관리 도구는 사용하기 번거로움
  - 여러 앱을 오가며 일정을 확인하는 것이 불편함
- **기대 효과**: 업무와 개인 일정을 한눈에 확인하고 효율적으로 관리

#### 페르소나 2: 김민정 (대학생)

- **나이/직업**: 20대 / 대학생
- **목표**: 수업, 과제, 시험 일정 관리
- **페인 포인트**:
  - 과제 마감일을 놓치는 경우가 많음
  - 학교 일정과 개인 일정을 함께 관리하기 어려움
- **기대 효과**: 학교 수업과 과제를 체계적으로 관리하여 학업 성취도 향상

## 3. 핵심 사용자 스토리

### 3.1 Must-Have (필수 기능)

1. **사용자로서, 나는 계정을 만들고 로그인하여 내 할 일을 관리하고 싶다.**

   - 회원가입, 로그인, 로그아웃 기능 제공
   - 안전한 인증 메커니즘 (JWT 기반)

2. **사용자로서, 나는 할 일을 추가/수정/삭제하여 내 일정을 관리하고 싶다.**

   - 할 일 생성, 조회, 수정, 삭제 (CRUD)
   - 제목, 설명, 시작일, 마감일, 완료 상태 관리

3. **사용자로서, 나는 실수로 삭제한 할 일을 복구할 수 있기를 원한다.**

   - 휴지통 기능 (소프트 삭제)
   - 휴지통에서 복구 또는 영구 삭제

4. **사용자로서, 나는 캘린더 형식으로 내 일정을 시각적으로 보고 싶다.**

   - 월별 캘린더 뷰
   - 국경일과 개인 할 일 통합 표시

5. **사용자로서, 나는 국경일을 함께 확인하여 휴일을 고려한 계획을 세우고 싶다.**
   - 국경일 자동 표시
   - 국경일과 개인 일정 구분 표시

### 3.2 Should-Have (중요 기능)

1. **캘린더 시각화 고도화**

   - 주간 뷰, 월간 뷰 전환
   - 날짜별 할 일 필터링

2. **할 일 검색 및 필터링**

   - 키워드 검색
   - 날짜별, 상태별 필터링
   - 우선순위, 날짜별 정렬

3. **사용자 프로필 관리**
   - 사용자 정보 조회 및 수정
   - 비밀번호 변경

### 3.3 Could-Have (선택 기능)

1. **알림 기능**: 마감일 임박 알림
2. **할 일 공유**: 특정 할 일을 다른 사용자와 공유
3. **통계 대시보드**: 완료율, 생산성 통계
4. **태그 기능**: 할 일 카테고리 분류
5. **반복 할 일**: 매일, 매주, 매월 반복되는 할 일 설정

## 4. 상세 기능 명세

### 4.1 사용자 인증 및 관리

#### 4.1.1 회원가입 (UC-01)

**기능 설명**: 신규 사용자가 계정을 생성할 수 있습니다.

**입력**:

- 사용자명 (username) - 필수, 고유값
- 이메일 (email) - 필수, 고유값
- 비밀번호 (password) - 필수, 최소 8자

**처리**:

- 사용자명 중복 확인
- 이메일 유효성 검증
- 비밀번호 암호화 (bcrypt 등)
- 데이터베이스에 사용자 정보 저장

**출력**:

- 성공: 회원가입 성공 메시지
- 실패: 중복 사용자명/이메일, 유효하지 않은 입력 등 오류 메시지

**비즈니스 규칙**:

- C-11: 사용자명은 고유해야 함
- C-01: 비밀번호는 암호화하여 저장

#### 4.1.2 로그인 (UC-02)

**기능 설명**: 등록된 사용자가 인증하여 시스템에 접근할 수 있습니다.

**입력**:

- 사용자명 또는 이메일
- 비밀번호

**처리**:

- 사용자 정보 조회
- 비밀번호 검증
- Access Token 및 Refresh Token 생성 (JWT)

**출력**:

- 성공: Access Token, Refresh Token 반환
- 실패: 인증 실패 메시지

**비즈니스 규칙**:

- C-04: JWT 기반 인증
- C-05: Access Token과 Refresh Token 함께 사용

#### 4.1.3 로그아웃 (UC-03)

**기능 설명**: 사용자가 시스템에서 로그아웃합니다.

**입력**:

- Access Token (헤더)

**처리**:

- 토큰 무효화 (블랙리스트 또는 만료 처리)

**출력**:

- 로그아웃 성공 메시지

#### 4.1.4 사용자 프로필 조회 및 수정

**기능 설명**: 사용자가 자신의 개인정보를 조회하고 수정할 수 있습니다.

**입력**:

- 조회: 없음 (인증된 사용자)
- 수정: 이메일, 비밀번호 등

**처리**:

- 사용자 정보 조회/수정
- 비밀번호 변경 시 재암호화

**출력**:

- 사용자 정보 또는 수정 성공 메시지

### 4.2 할 일 관리 (CRUD)

#### 4.2.1 할 일 목록 조회 (UC-04)

**기능 설명**: 사용자가 자신의 할 일 목록을 조회합니다.

**입력**:

- 인증된 사용자 정보
- 필터 옵션 (선택):
  - 날짜 범위
  - 완료 여부
  - 검색 키워드

**처리**:

- 현재 사용자의 할 일 조회 (isDeleted = false)
- 국경일 포함 (userId = null, isPublicHoliday = true)
- 필터 및 정렬 적용

**출력**:

- 할 일 목록 (배열)
  - todoId, title, description, startDate, dueDate, isCompleted, isPublicHoliday

**비즈니스 규칙**:

- BR-01: 로그인한 사용자만 접근 가능
- BR-06: 자신의 할 일과 국경일 함께 조회
- BR-10: 휴지통의 할 일은 제외

#### 4.2.2 할 일 상세 조회 (UC-05)

**기능 설명**: 특정 할 일의 상세 정보를 조회합니다.

**입력**:

- todoId

**처리**:

- 할 일 조회
- 권한 확인 (본인 소유 또는 국경일)

**출력**:

- 할 일 상세 정보

**비즈니스 규칙**:

- BR-02: 본인이 생성한 할 일만 접근 가능 (국경일 제외)

#### 4.2.3 할 일 추가 (UC-06)

**기능 설명**: 새로운 할 일을 생성합니다.

**입력**:

- title (필수)
- description (선택)
- startDate (선택)
- dueDate (선택)

**처리**:

- 입력 데이터 검증
- 현재 사용자를 소유자로 설정
- 데이터베이스에 저장

**출력**:

- 생성된 할 일 정보

**비즈니스 규칙**:

- BR-04: 제목은 필수
- BR-05: 과거 날짜도 입력 가능
- C-09: 제목 최대 200자
- C-10: 설명 최대 2000자
- C-12: 만료일은 시작일 이전일 수 없음

#### 4.2.4 할 일 수정 (UC-07)

**기능 설명**: 기존 할 일의 정보를 수정합니다.

**입력**:

- todoId
- 수정할 필드 (title, description, startDate, dueDate)

**처리**:

- 권한 확인 (본인 소유)
- 입력 데이터 검증
- 데이터베이스 업데이트

**출력**:

- 수정된 할 일 정보

**비즈니스 규칙**:

- BR-02: 본인이 생성한 할 일만 수정 가능
- C-15: 완료된 할 일은 수정 불가

#### 4.2.5 할 일 완료 상태 토글 (UC-08)

**기능 설명**: 할 일의 완료 여부를 변경합니다.

**입력**:

- todoId
- isCompleted (true/false)

**처리**:

- 권한 확인
- 완료 상태 업데이트

**출력**:

- 업데이트된 할 일 정보

#### 4.2.6 할 일 삭제 (UC-09)

**기능 설명**: 할 일을 휴지통으로 이동합니다 (소프트 삭제).

**입력**:

- todoId

**처리**:

- 권한 확인
- isDeleted를 true로 설정
- deletedAt에 현재 시간 기록

**출력**:

- 삭제 성공 메시지

**비즈니스 규칙**:

- BR-07: 즉시 제거되지 않고 휴지통으로 이동
- C-15: 완료된 할 일은 삭제 불가

### 4.3 휴지통 관리

#### 4.3.1 휴지통 항목 조회 (UC-10)

**기능 설명**: 삭제된 할 일 목록을 조회합니다.

**입력**:

- 인증된 사용자 정보

**처리**:

- isDeleted = true인 할 일 조회

**출력**:

- 삭제된 할 일 목록

**비즈니스 규칙**:

- BR-08: 휴지통의 할 일은 복구 또는 영구 삭제 가능

#### 4.3.2 할 일 복구 (UC-11)

**기능 설명**: 휴지통의 할 일을 복구합니다.

**입력**:

- todoId

**처리**:

- 권한 확인
- isDeleted를 false로 설정
- deletedAt을 null로 초기화

**출력**:

- 복구 성공 메시지

#### 4.3.3 할 일 영구 삭제 (UC-12)

**기능 설명**: 할 일을 데이터베이스에서 완전히 제거합니다.

**입력**:

- todoId

**처리**:

- 권한 확인
- 데이터베이스에서 물리적 삭제

**출력**:

- 영구 삭제 성공 메시지

**비즈니스 규칙**:

- BR-09: 영구 삭제된 할 일은 복구 불가

### 4.4 캘린더 뷰

#### 4.4.1 캘린더 형식으로 할 일 조회 (UC-13)

**기능 설명**: 할 일을 캘린더 형식으로 시각화하여 조회합니다.

**입력**:

- 연도 (year)
- 월 (month)

**처리**:

- 해당 월의 모든 날짜 생성
- 각 날짜에 해당하는 할 일 조회
- 국경일 포함

**출력**:

- 날짜별 할 일 정보
- 국경일 정보

**비즈니스 규칙**:

- BR-14: 개인 할 일과 국경일 함께 표시
- BR-15: 만료 기한이 없는 할 일은 표시되지 않음
- BR-16: 삭제된 할 일은 표시되지 않음

#### 4.4.2 특정 날짜의 할 일 조회 (UC-14)

**기능 설명**: 특정 날짜에 해당하는 할 일을 조회합니다.

**입력**:

- 날짜 (date)

**처리**:

- 해당 날짜를 dueDate로 가진 할 일 조회

**출력**:

- 해당 날짜의 할 일 목록

#### 4.4.3 월별 할 일 요약 조회 (UC-15)

**기능 설명**: 월별로 할 일 통계를 조회합니다.

**입력**:

- 연도 (year)
- 월 (month)

**처리**:

- 해당 월의 할 일 통계 계산
  - 전체 할 일 수
  - 완료된 할 일 수
  - 미완료 할 일 수

**출력**:

- 월별 통계 정보

### 4.5 국경일 관리

#### 4.5.1 국경일 조회 (UC-16)

**기능 설명**: 국경일 목록을 조회합니다.

**입력**:

- 연도 (선택)

**처리**:

- isPublicHoliday = true인 항목 조회

**출력**:

- 국경일 목록

**비즈니스 규칙**:

- BR-11: userId가 null인 공통 할 일
- BR-13: 모든 사용자에게 동일하게 표시

### 4.6 관리자 기능

#### 4.6.1 국경일 생성/수정/삭제

**기능 설명**: 관리자가 국경일을 관리합니다.

**입력**:

- 국경일 정보 (title, dueDate)

**처리**:

- 관리자 권한 확인
- 국경일 생성/수정/삭제

**출력**:

- 성공 메시지

**비즈니스 규칙**:

- BR-03: 관리자만 수정/삭제 가능
- BR-12: 일반 사용자 생성 불가

## 5. 성공 지표 (Success Metrics)

### 5.1 핵심 성공 지표 (KPI)

| 지표                               | 목표    | 측정 방법                 |
| ---------------------------------- | ------- | ------------------------- |
| **DAU (Daily Active Users)**       | 100명   | 일일 로그인 사용자 수     |
| **MAU (Monthly Active Users)**     | 3,000명 | 월간 로그인 사용자 수     |
| **사용자 유지율 (Retention Rate)** | 80%     | 30일 내 재방문율          |
| **할 일 완료율**                   | 50%     | 완료된 할 일 / 전체 할 일 |
| **평균 세션 시간**                 | 1분     | 평균 사용 시간            |

### 5.2 사용자 만족도 지표

- **사용자 피드백 및 설문조사**
  - 분기별 사용자 설문 실시
  - 피드백 수집 및 개선사항 도출

### 5.3 기술 지표

#### 시스템 아키텍처

```
[클라이언트 (React)]
        |
        | HTTPS
        v
[API 서버 (Express)]
        |
        | SQL
        v
[데이터베이스 (PostgreSQL)]
```

#### 인증 플로우

```
1. 사용자 로그인 요청
2. 서버: 사용자 인증
3. 서버: Access Token + Refresh Token 발급
4. 클라이언트: 토큰 저장 (localStorage/cookie)
5. 이후 요청: Access Token을 헤더에 포함
6. 서버: 토큰 검증 후 요청 처리
7. Access Token 만료 시: Refresh Token으로 갱신
```

## 7. 보안 및 개인정보

### 7.1 보안 요구사항

| 항목                   | 내용                              |
| ---------------------- | --------------------------------- |
| **HTTPS 사용**         | 모든 통신은 HTTPS를 통해 암호화   |
| **비밀번호 암호화**    | bcrypt를 사용하여 비밀번호 해싱   |
| **JWT 기반 인증**      | Access Token + Refresh Token 사용 |
| **CORS 정책**          | 허용된 도메인만 API 접근 가능     |
| **SQL Injection 방어** | Prepared Statement 사용           |
| **XSS 방어**           | 입력값 검증 및 이스케이프 처리    |
| **Rate Limiting**      | API 호출 제한을 통한 DDoS 방어    |

### 7.2 인증 및 권한

- **C-04, C-05**: JWT 기반 인증, Access Token + Refresh Token 사용
- **C-06**: 인증되지 않은 사용자의 API 접근 차단
- **C-07**: 사용자는 타인의 할 일에 접근 불가

### 7.3 데이터 보호

- **비밀번호**: bcrypt로 암호화 후 저장
- **개인 정보**: 최소한의 정보만 수집 (username, email)
- **데이터 백업**: 미정 (Supabase 자동 백업 활용)

### 7.4 규정 준수

- **개인정보보호법**: 사용자 동의 하에 정보 수집 및 처리
- **GDPR**: 향후 유럽 사용자 대상 시 준수 검토

## 11. 부록

### 11.1 용어 정의

| 용어                          | 정의                                                 |
| ----------------------------- | ---------------------------------------------------- |
| **할 일 (Todo)**              | 사용자가 완료해야 하는 작업이나 일정                 |
| **개인 할 일**                | 특정 사용자가 생성하고 소유한 할 일                  |
| **공통 할 일**                | 모든 사용자에게 공통으로 표시되는 할 일 (국경일 등)  |
| **국경일 (Public Holiday)**   | 법정 공휴일 또는 국가 기념일                         |
| **휴지통 (Trash)**            | 삭제된 할 일이 임시로 보관되는 논리적 공간           |
| **소프트 삭제 (Soft Delete)** | 데이터를 물리적으로 삭제하지 않고 삭제 플래그만 설정 |
| **영구 삭제 (Hard Delete)**   | 데이터를 데이터베이스에서 물리적으로 완전히 제거     |
| **만료 기한 (Due Date)**      | 할 일을 완료해야 하는 목표 날짜                      |
| **MVP**                       | Minimum Viable Product (최소 기능 제품)              |
